# Intent

The Jaros OS must boot once and keep running — a persistent, observable node that hosts the harness, the durable replayable state machine, and the lightweight agent-thread pool, and continuously turns incoming work into validated, durable state transitions. Work and new agents enter ONLY through the shared file system (an inbox of jobs and a plugins folder), never a network channel. The OS requires no external server, database, or message broker to run — only files and threads. A single agent or job failure is contained; the OS stays up. This is what makes Jaros a runnable operating system for agents rather than a one-shot script — the standing embodiment of the Prime Directive's state-machine backbone (REQ-2), thread runtime (REQ-3), governing harness (REQ-5), and shared-FS communication (REQ-6).
