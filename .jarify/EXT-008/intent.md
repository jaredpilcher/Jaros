# Intent

An operator on any host — Windows, macOS, or Linux — must be able to add work and new agents to a running Jaros OS, and watch it, through one cross-platform command-line interface. The CLI communicates with the OS EXCLUSIVELY through the shared file system (a mounted volume): it writes job files to the inbox, installs agent modules into the agents folder, and reads results and live status back. No sockets, no HTTP, no platform-specific calls — pure files. This keeps the host↔OS control plane faithful to the Prime Directive's communication tenet (shared FS + queues only, REQ-6) and trivially portable across operating systems.
